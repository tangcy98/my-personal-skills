# 任务指令模板

## 通用格式

```
任务：[一句话核心目标]

背景：[为什么做，给谁用，解决什么问题]

输入：
- [材料1：文件/数据/链接]
- [材料2]

要求：
- [关键约束1]
- [关键约束2]
- [输出形式/标准]

验收：[怎么算完成]
```

## 场景模板

### 代码开发

```
任务：[动作] [目标]

背景：[问题现象/优化目标]

输入：
- 现有代码：[路径/内容]
- 相关依赖：[版本/框架]

要求：
- [功能/性能要求]
- [代码规范]
- [测试要求]

验收：
- [ ] 功能正确
- [ ] 性能达标
- [ ] 测试通过
```

**常见建议**：
- 复杂逻辑先写伪代码或流程图
- 关键路径加单元测试
- 考虑边界情况和异常处理

### 数据分析

```
任务：分析 [数据对象] 输出 [成果]

背景：[分析目的/决策支持]

输入：
- 数据源：[路径/API]
- 数据说明：[字段含义]

要求：
- [分析维度]
- [关键指标]
- [排除项]
- [输出形式]

验收：
- 数据准确
- 结论有据
- 可视清晰
```

**常见建议**：
- 先探查数据质量（缺失值、异常值）
- 明确分析维度后再深入
- 可视化前确定核心结论，图表服务于观点

### 文档/内容

```
任务：[撰写/整理/翻译] [内容类型]

背景：[目标读者/使用场景]

输入：
- 原始材料：[来源]
- 参考资料：[如有]

要求：
- [风格/语气]
- [格式要求]
- [长度范围]

验收：[质量检查点]
```

**常见建议**：
- 先列大纲再填充内容
- 每段一个核心观点
- 技术文档多用图表和代码示例

### 研究/调研

```
任务：调研 [对象] 聚焦 [范围]

背景：[研究目的]

输入：
- 已知信息：[已有结论]
- 资源约束：[时间/预算]

要求：
- [信息源要求]
- [分析深度]
- [输出结构]

验收：
- 来源可靠
- 分析严谨
- 建议可行
```

**常见建议**：
- 先界定范围，避免过度发散
- 多源交叉验证关键信息
- 结论要有数据或案例支撑

## 设计原则

- 简洁 > 详细
- 约束 > 步骤（让 Agent 自己规划怎么做）
- 目标明确 > 过程完备
